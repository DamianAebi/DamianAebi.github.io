Ein Programm ist eine Beschreibung einer Lösung.

Sie wird in einer bestimmten Programmiersprache geschrieben (source Code).

EVA: Eingabe, Verarbeitung, Ausgabe (z.B. Taschenrechner aber auch Programme)

In der Informatik kommt dann die Speicherung auch noch dazu

Computing: auf algorithmischen Prozessen aufbauend, z.B. bei der Programmierung / bei der Entwicklung von Software / Hardware

Computational thinking: so zu denken, dass die Lösung von Mensch und Maschine verstanden wird

Variable: hat Namen und einen variablen (veränderbaren) Wert. Dieser Wert ist hardwaretechnisch gesehen nicht direkt bei der Variable. Die Variable hat lediglich die Adresse zum Block im RAM, welcher den Wert speichert.

Die Modellierung- und Problemlösungsprozesse sind dabei von einer Programmiersprache unabhängig.

Gute Programmierung

  • code ist korrekt
  • code ist robust
  • code ist zuverlässig
  • code hat einfache Wartbarkeit

Guter Programmierstil

  • Der Quellcode ist selbsterklärend
  • Variablen und Funktionennamen sind intuitiv verständlich
  • Quellcode ist kommentiert
  • Konventionen der Programmiersprache müssen eingehalten werden
  • DRY (don’t repeat yourself)
  • KISS (keep it simple stupid)

Generationen

  • Maschinencode (0 und 1)
  • Assemblersprachen (sind gebunden an Hardware und werden auch überwiegend zur Programmierung von Hardware gebraucht)
  • Prozedurale Sprache (sind Systemunabhängig, leicht zu verstehen)
  • 4th Generation Language (4GL) (Sagen, was für ein Resultat man will, aber nicht wie es gemacht werden muss, z.B. SELECT query)
  • Künstliche Intelligenz

Um eine Sprache in Maschinencode umzuwandeln, braucht man einer dieser drei Varianten

  • Interpreter (Übersetzung während der Programmausführung)
  • Compiler (Alles wird auf einmal vor der Programmausführung kompiliert)
    • Alles muss richtig sein, dass man kompilieren kann
    • es geht länger
    • Ausführung ist jedoch schneller
  • Zweistufige Übersetzung (Java, C#)
    • Übersetzung erfolgt über virtuelle Maschine (installiert auf dem eigenen PC)

Sprachen

  • Prozedural / funktional (Funktion kann andere Funktion aufrufen und Ergebnis nutzen)
  • Objektorientierte Sprachen (OOP) (Hat Klassen, z.B. C#, C++, Java)
  • Hybride Sprachen (können funktional/prozedural oder objektorientiert sein)
  • Erziehungsorientierte Sprachen (Sprachen, die nur zu Lernzwecken erstellt wurden, z.B. logo)

Bits und Bytes

  • Ein Byte hat 8 Bits
  • Ein Byte kann nur ein Zeichen speichern (z.B. den Buchstaben A)
  • 1 MB = 1000 KB
  • 1MiB = 1024 KiB

Hexadezimale Schreibweise

  • Binärer wert in 4er-Pakete umwandeln
  • dann jedes Paket ausrechnen
  • 1-9 sind normal, 10-16 sind Buchstaben (ABCDEFG)
  • z.B. 1011 = 11 = B

Oktale Schreibweise

  • Binärer Wert wird in 3er-Pakete umwandeln
  • z.B. 101101 = 101 & 101 = 5 & 5 = 55

Grafische Darstellung von Programmabläufen

  • Oval ist Beginn / Ende
  • Rechteck ist Geschehnis, z.B. Mausclick
  • Raute ist Entscheidung (z.B. x>6 ja / nein?)
