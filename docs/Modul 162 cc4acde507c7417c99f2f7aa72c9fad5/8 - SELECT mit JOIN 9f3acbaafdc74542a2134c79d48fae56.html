8 - SELECT mit JOIN

SELECT

<Spaltenliste>

FROM <linke Tabelle> AS <Alias>

[ <Join-Typ> ] JOIN <rechte Tabelle> AS <Alias>

ON <Bedingung>

WHERE...;

SELECT *
  FROM Lernende AS l
  INNER |LEFT | RIGHT JOIN Orte AS o
  ON l.ort_id = o.id;

  • AS definiert ein alias (soz. eine Variable) für die Auswahl und kann später im “ON”
  • Join-Typ
    • INNER
    • LEFT
    • RIGHT
  • ON
    • Nach dem Schlüsselwort ON wird die Verknüpfung vom Fremdschlüssel der einen Tabelle (Lernende) mit dem Primärschlüssel der anderen Tabelle (Orte) aufgeführt.

      Beispiel: ON l.ort_id = o.id

    Inner Join

    INNER JOIN ist die Schnittmenge der beiden Tabellen. Es werden nur diejenigen Datensätze zurückgeliefert, die in beiden Tabellen vorkommen.

    SELECT l.Vorname, l.Nachname, l.Strasse, o.PLZ, o.Ort

    FROM Lernende AS l

    INNER JOIN Orte AS o

    ON l.ort_id = o.id;

    Retourniere nur die Datensätze aus der Tabelle Lernende, in denen ein Fremdschlüsselwert ort_id vorhanden ist und ergänze die Ausgabe mit der Postleitzahl und dem Ort aus der Tabelle Orte.

    Verbund mit 3 Tabellen:

    SELECT l.Vorname, l.Nachname, l.Strasse, o.PLZ, o.Ort, f.Firma

    FROM Lernende AS l

    INNER JOIN Orte AS o

    ON l.ort_id = o.id

    INNER JOIN Firmen AS f

    ON l.firma_id = f.id;

    Left Join (aka LEFT OUTER JOIN)

    Bedeutung

    Beim LEFT JOIN werden alle Datensätze der linken Tabelle zurückgegeben.

    Definition

    LEFT JOIN gibt die linke Tabelle vollständig aus und von der rechten Tabelle nur die Datensätze, welche die ON-Klausel erfüllen.

    Testen Sie folgenden Befehl:

    SELECT l.Vorname, l.Nachname, l.Strasse, f.Firma

    FROM Lernende AS l

    LEFT JOIN Firmen AS f

    ON l.firma_id = f.id

    ORDER BY l.Vorname;

    Ausgesprochen

    Retourniere alle Datensätze der linken Tabelle Lernende, egal ob ein Fremdschlüsselwert firma_id vorhanden ist oder nicht. Falls ein Fremdschlüsselwert vorhanden ist: Ergänze die Ausgabe mit dem Firmennamen aus der Tabelle Firmen.

    LEFT Excluding JOIN

    Spezialfall, wenn nur jene Einträge gewünscht sind, welche zu Tabelle B keinen Match aufweisen

    SELECT *
FROM tableA a LEFT OUTER JOIN tableB b
ON a.name = b.name
WHERE b.name IS NULL;
    +------+-----------+------+------+
| id   | name      | id   | name |
+------+-----------+------+------+
|    2 | Monkey    | NULL | NULL |
|    4 | Spaghetti | NULL | NULL |
+------+-----------+------+------+

    Right Join (aka RIGHT OUTER JOIN)

    Right Joins sind im SQLite nicht möglich. Man muss deswegen einen LEFT-join machen und die Reihenfolge der Tabellen vertauschen.

    In MySQL hingegen, kann man folgendes schreiben:

    SELECT * FROM tableA a RIGHT JOIN tableB b
ON a.name = b.name;

    Right excluding Join

    Auch hier ist der Spezialfall möglich, wenn nur jene Einträge gewünscht sind, bei welchen keine Matches auf der linken Tabelle vorhanden sind.

    SELECT *
FROM tableA a RIGHT OUTER JOIN tableB b
ON a.name = b.name
WHERE a.name IS NULL;
    +------+------+------+-------------+
| id   | name | id   | name        |
+------+------+------+-------------+
| NULL | NULL |    1 | Rutabaga    |
| NULL | NULL |    3 | Darth Vader |
+------+------+------+-------------+

    Full outer Join

    Ebenfalls nicht implementiert, kann jedoch mit einem UNION von LEFT und RIGHT Joins erschaffen werden

    SELECT l.Vorname, l.Nachname, l.Strasse, o.PLZ, o.Ort

    FROM Lernende AS l

    LEFT JOIN Orte AS o

    ON l.ort_id = o.id

    UNION SELECT l.Vorname, l.Nachname, l.Strasse, o.PLZ, o.Ort

    FROM Orte AS o

    LEFT JOIN Lernende AS l

    ON l.ort_id = o.id

    ORDER BY Ort;

    Outer Excluding JOIN

    SELECT *
FROM tableA a LEFT OUTER JOIN tableB b
ON a.name = b.name
WHERE b.name IS NULL
UNION
SELECT *
FROM tableA a RIGHT OUTER JOIN tableB b
ON a.name = b.name
WHERE a.name IS NULL;
    +------+-----------+------+-------------+
| id   | name      | id   | name        |
+------+-----------+------+-------------+
|    2 | Monkey    | NULL | NULL        |
|    4 | Spaghetti | NULL | NULL        |
| NULL | NULL      |    1 | Rutabaga    |
| NULL | NULL      |    3 | Darth Vader |
+------+-----------+------+-------------+

    Union

    Mit UNION können zwei Ausgaben zusammengefügt werden. Voraussetzung: Die beiden Attributlisten müssen identisch sein.

    Beispiel: Wir wollen alle Adressen inkl. Ort von den Lernenden und von den Firmen in einer Ausgabe anzeigen. Die Liste soll nach 1. Ort und 2. Strasse sortiert werden.

    SELECT Strasse, PLZ, Ort FROM Lernende l

    JOIN Orte o ON l.ort_id = o.id

    UNION SELECT Strasse, PLZ, Ort FROM Firmen f

    JOIN Orte o ON f.ort_id = o.id

    ORDER BY Ort, Strasse;

    Self-Join

    Wenn man dieselbe Tabelle mit sich selber JOINED, nennt man dies ein “Self-Join”

    SELECT e.last_name AS 'Employee', m.last_name AS 'His Boss'
FROM employee e INNER JOIN employee m
ON e.manager_id = m.employee_id
ORDER BY 'His Boss', 'Employee';

    Cross Join

    Verbindet jedes Element der linken Tabelle mit jedem Element der rechten Tabelle. Dies führt zu x * y Verbindungen, wenn die linke Tabelle x Elemente hat und die Rechte y.

    Es ist daher aus Performance-Gründen unerwünscht.

    SELECT country_name, region_name 
FROM country JOIN region;
    | country_name             | region_name            |
+--------------------------+------------------------+
| Argentina                | Europe                 |
| Argentina                | Americas               |
| Argentina                | Asia                   |
| Argentina                | Middle East and Africa |
| Australia                | Europe                 |
| Australia                | Americas               |
| Australia                | Asia                   |
| Australia                | Middle East and Africa |
| Belgium                  | Europe                 |
| Belgium                  | Americas               |
| Belgium                  | Asia                   |
...
