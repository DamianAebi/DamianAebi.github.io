5 - SQL SELECT

SQL ist eine Datenbanksprache zur Definition, Abfrage und Manipulation von Daten in Relationalen Datenbanksystemen.

Es ist momentan laut TIOBE die 10.-meistverwendete Sprache der Welt.

Wird verwendet für

  • Datenmanipulation
  • Datenabfrage
  • Datendefinition (Befehle zur Definition des Datenbankschemas, z.B. Tabellen löschen, definition von Primärschlüssel etc.)
  • Datenrechtsverwaltung
  • Transaktoinskontrolle

In diesem Modul beschäftigen wir uns mit der Datenabfrage.

Select

SELECT

[ DISTINCT | ALL ]

<Spaltenliste>

FROM <Tabelle>

[ WHERE-Klausel ]

[ GROUP BY-Klausel ]

[ HAVING-Klausel ]

[ UNION [ALL] SELECT-Befehl ]

[ ORDER BY-Klausel ];

Mindestens erforderlich sind: Das Schlüsselwort SELECT, eine Liste von Spalten (<select list>), das Schlüsselwort FROM und ein Tabellenname (<table reference list>).

Beispiel: SELECT Vorname, Nachname, Jahrgang FROM Lernende;
Es werden Vorname, Nachname und Jahrgang von allen Lernenden ausgegeben.

LIMIT

Wenn nur eine bestimmte (bzw. maximale) Anzahl von Zeilen ausgegeben werden soll, kann die LIMIT-Klausel ans Ende des SELECT-Befehls gesetzt werden.

SELECT Vorname, Nachname, Geburtsdatum FROM Lernende LIMIT 10;

AS

Mit AS kann sowohl ein Spaltentitel geändert werden als auch ein Alias für den Tabellennamen erstellt werden.

SELECT Vorname || ' ' || Nachname AS 'Name', Strasse, Plz, Ort FROM Lernende;

Berechnungen

Man kann bei numerischen Attributen Berechnungen im SELECT verwenden:

SELECT Artikelname, Preis * 1.2 FROM Artikel;

ORDER BY

Mit der ORDER BY-Klausel kann die Ausgabe sortiert werden, was in vielen Fällen Sinn macht.

SELECT * FROM Lernende ORDER BY Ort, Nachname;

  • > Es werden alle Lernenden ausgegeben, und zwar sortiert: In erster Linie nach Ort und in zweiter Linie nach Nachnamen (in den Fällen, wo es mehrere Lernende pro Ort gibt).
  • man kann auch ASC (ascending, aufsteigend) und DESC (descending, absteigend) dahinter setzen um die Reihenfolge zu definieren

Sämtliche Operatoren und Bedingungen können sowohl bei alphanumerischen (Zeichenketten) als auch numerischen (Zahlen) Wertebereichen angewendet werden.

Beispiel:

SELECT * FROM lernende WHERE jahrgang = 2005

SELECT * FROM lernende WHERE nachname BETWEEN 'D' AND 'N’

Platzhalter

Windows / LinuxSQLBedeutung
*%Platzhalter für beliebige Zeichen, z.B. "M%" = alles, was mit M beginnt
?_(Underscore)Platzhalter für ein Zeichen, z.B. "Ju_er" = die Nachnamen "Juker" und "Jufer" (und allenfalls weitere Nachnamen)

Platzhalter können ausschliessliche mit der LIKE-Klausel verwendet werden.

SELECT * FROM Lernende WHERE Nachname LIKE "M%";

Alle Lernenden, deren Nachname mit "M" beginnt, werden ausgegeben.

Korrekte Abfrage auf leere Werte

Da wir in der Regel nicht wissen, ob in den Daten leere Zeichenketten oder NULL-Werte vorhanden sind, sollten wir Abfragen immer wie folgt formulieren:

SELECT * FROM Lernende WHERE Firma = "" OR Firma IS NULL;

GROUP BY

Wenn wir alle Wohnorte der Lernenden ausgeben wollen, können wir folgende Abfrage verwenden:

SELECT DISTINCT Ort FROM Lernende;

Eine zweite Möglichkeit, die dasselbe Ergebnis liefert, ist folgende Abfrage:

SELECT Ort FROM Lernende GROUP BY Ort;

Was ist der Sinn von GROUP BY (warum kompliziert, wenn es auch einfacher geht)? Diese Klausel wird v.a. dafür verwendet, gruppenweise Auswertungen durchzuführen (Aggregatsfunktion), was mit DISTINCT nicht möglich ist.

SELECT Ort, AVG(2021-Jahrgang) AS "Durchschnittsalter" FROM Lernende GROUP BY Ort;

  • > Das durchschnittliche Alter der Lernenden pro Ort wird ausgegeben.

Es ist auch möglich, nach einem Paar von Attributen zu gruppieren: 

SELECT blockfloor Floor, blockcode as Block, Count(*) as "Number of available rooms" 
FROM room 
WHERE unavailable = 'f' 
GROUP BY blockfloor, blockcode 
ORDER BY blockfloor, blockcode;

HAVING

Mit having kann bei GROUP BY zusätzlich ein Filter gesetzt werden. SELECT Ort, AVG(2021-Jahrgang) AS "Durchschnittsalter" FROM Lernende GROUP BY Ort HAVING Ort > "Berlin";

Sortierung von Zeichenketten

  • numerische Sortierung
    • 1, 2, 3, 4, 5, 10, 20, 30, 100, 200
    • → Sortierung nach Grösse der Zahl
  • alphanumerische Sortierung
    • “1”, “10”, “100”, “2”, “20”, “200”, “3”, “30”, “4”, “5”
    • → Sortierung nach ASCII Tabelle
    • wobei bei gleichen Anfängen zweier Zahlen die Kürzere gewählt wird
