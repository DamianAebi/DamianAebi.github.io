Frameworks

  • Frameworks sind eine Sammlung von Tools
  • Sie vereinfachen die Arbeit, also man kann mit weniger Arbeit ein Resultat erreichen
  • Man kann effizienter arbeiten
  • Es gibt ausserdem Konsistenz im Code
  • Es ist kompatibel mit Browsern
  • Ausserdem sind die Elemente schon “getested”, man wird also allgemein in weniger Probleme stossen

CSS Frameworks

  • Pure CSS
    • sehr minimal
  • Tailwind CSS
    • grosse Auswahl an Komponenten
  • Bootstrap
    • Eines der ersten responsive-Frameworks
    • arbeited mit Grid

Auftrag Präsentation (bis 11 Uhr Zeit):

  • Szenario: Kunde will Framework, welches effizient und anhaltend ist
  • Gewähltes Framework: Bootstrap

  • Warum sollte dieses Framework verwendet werden? (Unique selling point)
    • Schnelle Entwicklung: Bootstrap bietet eine Vielzahl vorgefertigter HTML- und CSS-Komponenten, die es Entwicklern ermöglichen, schnell und effizient Webseiten zu erstellen. Dies führt zu einer beschleunigten Entwicklungszeit.
    • Mobile- und Browser-Kompatibilität: Bootstrap ist von Grund auf für responsives Webdesign konzipiert. Mit dem Grid-System und den responsiven Klassen können Entwickler problemlos Websites erstellen, die auf verschiedenen Bildschirmgrößen und -geräten gut aussehen.
      • Bootstrap wurde getestet und ist in den meisten gängigen Webbrowsern stabil. Dies erleichtert die Sicherstellung einer konsistenten Benutzererfahrung über verschiedene Plattformen hinweg.
    • Anpassbarkeit: Obwohl Bootstrap vorgefertigte Stile und Komponenten bietet, ist es auch sehr anpassbar. Entwickler können das Framework erweitern und eigene Stile und Komponenten hinzufügen, um ihren speziellen Anforderungen gerecht zu werden.
  • Für welche Use-Cases ist es besonders geeignet?
    • Start-ups und kleine Unternehmen: Bootstrap ist eine kostengünstige Option für Start-ups und kleine Unternehmen, die keine umfangreichen Ressourcen für die Frontend-Entwicklung haben. Es ermöglicht die Erstellung ansprechender Websites mit begrenztem Aufwand.
    • Bei komplexen, stark maßgeschneiderten Projekten oder Anwendungen mit besonderen Anforderungen könnten andere Frameworks oder sogar eine maßgeschneiderte Entwicklung erforderlich sein.
  • Wie beliebt ist das Framework?
    • [In 2023] Bootstrap is used by 25.8% of all the websites whose JavaScript library we know. This is 21.1% of all websites.
      • https://w3techs.com/technologies/details/js-bootstrap
    • [In 2018,] Overall, BuiltWith shows around 18 million websites using Bootstrap
      • https://ostraining.com/blog/webdesign/bootstrap-popular/
  • Wie alt ist das Framework?
    • Es gibt es schon seit dem 19. August 2011, also etwa 12 Jahren
    • Es ist eines der bekanntesten CSS-Frameworks überhaupt und die Community ist aktiv
    • Version 5.3.1 wurde Ende Juli dieses Jahres veröffentlicht
  • Würdest du dieses Framework verwenden, um für Kunden und für dich eine Webseite zu erstellen
    • Es kommt immer auf das Projekt darauf an. Aber für simple und effiziente Projekte, ist es eine sehr gute Wahl.
  • Euer Fazit zum Framework
    • Ist gut zu kennen und bietet einen guten Einstieg in die Welt der Frameworks.

Tailwind

  • hat verschiedene Styles (z.B. dark, bright mode)
  • arbeited mit JS Frameworks wie (Vue, JS, Angular)
  • auch nur für einfachere Webseiten (braucht viel Speicherplatz, was in Ladezeiten resultiert)

Pure CSS

  • Leichtgewichtig
    • hat minimale Dateigrösse für gute Performance
  • Responsive Design
  • Modularität
    • man kann nur die Teile verwenden, welche man benötigt
  • Meisbekannt in der USA
  • Problem ist, dass es nicht oft upgedated wird

Für grössere Projekte eignen sich JS-Frameworks wie Angular oder React besser.
