Positionierung

  • Es gibt Block-Elemente
    • sie nehmen standard-mässig die ganze Breite der Seite an und schieben die folgenden Tags nach unten (unabhänging von der tatsächlichen Breite des Elements, also auch Block-Elemente mit 200px Breite machen das)
    • z.B. div, nav, …
  • Und Inline-Elemente
    • sie nehmen nur die nötige Breite an und erlauben so, dass mehrere inline-Elemente nebeneinander dargestellt werden können
    • z.B. strong, a, …

Box-Modell

  • Wenn man einem Element sagt, es soll display:inline sein, dann funktioniert das Box-Modell dabei nicht mehr, es nimmt nur die Breite an, welche nötig ist
  • Man müsste also display:inline-block angeben
  • margin kann z.B. so angegeben werden
p {
	margin-left: 20px  /* nur links Margin */
	margin: 20px /* überall das selbe Margin */
	margin: 10px 20px 10px 20px /* oben und unten 10px, rechts und links 20, t r b l */
	margin: 10px 20px /* ist das selbe */
	margin: auto; /* zentriert, funktioniert jedoch nicht bei inline-elementen (margin: auto bleibt 0)' */
}

position

  • static
    • standard-Wert
  • Elemente nehmen die ganze Breite ein
  • relative
    • Elemente werden von ihrem Normalwert verschoben
    • der ursprüngliche Platz den sie einnahmen, bleibt trotzdem reserviert
  • absolute
    • werden positioniert relativ zum Rand des nächsten übergeordneten Elementes, welches nicht position: static hat. Falls keines vorhanden ist, wird das Wurzelelement html verwendet.
    • Elemente werden aus dem Elementfluss herausgenommen, dass heisst der Platz, welchen sie ursprünglich reserviert haben, ist nicht mehr reserviert
  • fixed
    • wird relativ zum ViewPort platziert, bleibt also beim scrollen stehen
  • sticky
    • wird relativ zum ViewPort positioniert, jedoch wird es beim scrollen diesmal mitgehen mit den anderen Elementen, bis es den rand des ViewPorts erreicht

Display

  • block
    • macht Element zu einem Block-Element
  • inline
    • Elemente nehmen nur benötigten Platz
    • Elemente können nun nebeneinander angezeigt werden
    • verlieren nun das Box-Modell
  • inline-block
    • gleich wie inline, jedoch hat es immernoch das Box-Modell
  • none
    • Element verschwindet komplett (belegt auch keinen Platz, anders visibility:hidden)

Flexboxen

  • Hat Flex-Box-Container als Eltern-Element (display:flex)
  • Dann hat dieses Child-Elemente, welche sich in Richtung der “Flex-Direction” ausrichten (main-axis), und sich darauf verteilen
  • flex-direction: column, ordned die Elemente vertikal an
  • mit row-reverse / column-reverse kann man die Reihenfolge umdrehen
  • justify-content
    • flex-start : Elemente werden am start angeordned, ohne space
    • flex-end : hier am ende
    • center : einmitten
    • space-between : alle Elemente haben den gleichen Space um sich herum (äusserste elemente haben weniger space ganz aussen)
    • space-evenly : überall der gleiche space
  • align-items
    • Ausrichtung auf der Cross-Axis (also nicht Flex-Direction)
    • flex-start
    • flex-end
    • center
    • stretch
