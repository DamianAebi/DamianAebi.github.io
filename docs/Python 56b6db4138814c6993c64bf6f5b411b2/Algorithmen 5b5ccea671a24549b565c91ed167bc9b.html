Algorithmen

Eine endliche Abfolge von Instruktionen zum Lösen eines Problems

Algorithmen sind ein Set von Instruktionen. Es muss nicht unbedingt ein Programm sein. Es könnte auch ein Kochrezept sein.

Teilen grundsätzlich folgende Eigenschaften:

  • Input
  • Output
  • Eindeutig (Schritte sind klar)
  • Korrekt (Output ist korrekt)
  • Endlich (Die Anzahl Schritte soll endlich sein)
  • Effektiv (Jeder Schritt in endlicher Zeit erledigt)
  • Generell (sollte wieder verwendbar sein)

Damit ein Algorithmus zur Lösung einer Aufgabe in Programmen eingesetzt werden kann, muss er bestimmte Eigenschaften besitzen.

  • Endlich: Der Algorithmus muss vollständig mit einer endlichen Anzahl von Anweisungen beschrieben werden können
  • Allgemein: Der Algorithmus muss für die gleichen Eingabedaten immer das gleiche Ergebnis liefern
  • Eindeutig: Jeder Schritt des Algorithmus muss eindeutig ausgeführt werden können
  • Terminiert: Der Algorithmus muss nach einer endlichen Anzahl von Schritten zum Ende kommen

Algorithmen können nach verschiedenen Kriterien unterschieden werden, z. B. nach der Art der Abarbeitung oder dem Umgang mit Datentypen.

Suchalgorithmen

  • Können linear sein (geht über jedes Element, bis gefunden)
  • oder binär
    • geht nur wenn die liste sortiert ist
    • Beispiel: man sucht eine Zahl zwischen 1 und 100 und jedes mal bekommt man einen Tipp, ob die geschätzte Zahl zu hoch oder zu tief ist. Nun braucht man maximal log(2)100 Versuche (zuerst Tippt man 50, dann z.B. 75, usw. …)

Sortieralgorithmen

  • Bubblesort
    • jedes Element frägt das nächste, ob es grösser ist als das nächste. Wenn ja tauschen sie Plätze. Wenn nein, geht es mit dem nächsten weiter. Dies geht solange, bis keine Plätze mehr getauscht werden
  • Insertionsort
    • Passiert mit zwei listen, statt mit einer
    • Am Anfang ist die erste liste voll, die zweite leer
    • danach wir ein zufälliges Element in die zweite liste übergetragen
    • danach ein zweites, welches das erste fragt, ob es grösser ode kleiner ist
      • je nach je, geht es dann links oder rechts
    • danach kommt ein drittes, fängt links an und frägt erneut usw.
    • Ist vorallem bei kleineren Mengen von Listenelementen schneller als der Bubble-Sort
    • Muss jedoch eine zusätzliche Liste generieren
  • Merge sort
    • Divide and conquer
    • Array wird in kleinere Arrays gesplittet. Danach werden sie nach Bubble-Sort sortiert. Anschliessend werden sie wieder zusammengefügt

Iterativer Algorithmus

Ein iterativer Algorithmus löst eine Aufgabe mit Schleifen-Durchläufen.

# Berechnung der Fakultät mit einem iterativen Algorithmus# Die Fakultät ist in der Mathematik eine Funktion, die einer natürlichen Zahl das Produkt aller natürlichen Zahlen (ohne Null) kleiner und gleich dieser Zahl zuordnet.

def comp_fact(x):
    result = 1
    while x > 1:
        result = result * x
        x = x - 1
    return result


print(comp_fact(4))

Rekursiver Algorithmus

Die meisten Programmiersprachen unterstützen sogenannte rekursive Funktionsaufrufe – auch Selbstaufrufe genannt.

Wenn eine Funktion sich selber aufruft, muss dies, wie bei einer Schleife, durch eine Bedingung terminiert sein, dass sie nicht endlos läuft.

Wenn in einer Funktion dieselbe Funktion erneut aufgerufen wird, stoppt der Code und geht mit dem neuen Aufruf weiter.

Dies geht weiter und weiter so. Jedesmal, wenn die Funktion bei der Rekursion angelangt, geht sie einen Schritt tiefer.

Es entsteht ein Stack, mit dem LIFO Prinzip.

Es ist also der “tiefste” Schritt, welcher als erstes ausgeführt wird und der erste Funktionsaufruf wird somit als letztes beendet. 

# Berechnung der Fakultät mit einem rekursiven Algorithmus

def comp_fact(x):
    if x > 0:
        return x * comp_fact(x - 1)
    else:
        return 1


print(comp_fact(4))

Iterativ oder rekursiv?

Die iterativen und rekursiven Algorithmen spielen in der Programmierung eine wesentliche Rolle, da viele Probleme je nach Anforderung entweder durch Iteration oder Rekursion gelöst werden können.

Kurz gesagt:

  • Rekursive Lösungen können schneller entwickelt werden, da der Algorithmus nicht komplex wird und somit übersichtlicher ist
  • Iterative Lösungen enthalten in den meisten Fällen einen komplexen Algorithmus, können jedoch leichter optimiert werden (Performance)

Generischer Algorithmus

Ein algorithmus, welcher auf bestimmte Datentypen ausgelegt ist. Er funktioniert z.B. nur mit int oder string.

Beispiel Rekursiver Algorithmus

number_to_collatz = 13


def collatz(number, step):
    if number == 1:
        return step
    elif number % 2 == 0:
        print("number is ", number / 2)
        step += 1
        return collatz(number / 2, step)
    else:
        print("number is ", number * 3 + 1)
        step += 1
        return collatz(number * 3 + 1, step)


print("Start value is", number_to_collatz)
print("reached 1 in ", collatz(number_to_collatz, 0), " steps")
  • collatz-funktion
  • Achtung: hier müssen alle rekursiven Funktionsausführungsbefehle ein return vornedran haben. Nur zu returnen wenn die Zahl 1 ist funktioniert nicht, weil man die Zahl dann nur ein Rekursives Level nach oben returned, aber nicht bis ganz nach oben zum print.
