Klassen

  • Prozedurale Programmierung
    • Fokus auf Prozess
    • Es ist jedoch schwer zu erweitern oder zu verändern
  • Klassen/Objektbasiert
    • Klassen werden erstellt mit Attributen, Methoden
    • Klassen sind wie Baupläne und man kann mit ihnen Objekte erstellen
    • Objekte sind komplexe Datenstrukturen wie eg Arrays

Syntax:

class Person:
	# __init__ Funktion ist die Konstruktorfunktion. (Sie kann auch weggelassen werden und 
	# würde in dem Fall trotzdem im Hintergrund "unsichtbar" und leer laufen.) 
	# Sie wird automatisch ausgeführt. Self ist dabei ein Platzhalter für die Instanz
	def __init__(self, name):
		self.name = name

person1 = Person("Martin")
person2 = Person("Daniela")

Man sollte nicht manuell bei Instanzen Dinge hinzufügen. Nur was die Klasse zur Verfügung stellt. 

# nicht so
person1.height= 1.78

Falls man etwas hinzufügen will, was die Klasse nicht unterstützt, sollte man die Klasse ändern. 

class Person:
	def __init__(self, name, height):
		self.name = name
		self.height = height

person1 = Person("Martin", 1.78)
person2 = Person("Daniela", 1.65)

Methoden

Methoden sind Funktionen in einer Klasse. z.B.

class Person:
	def __init__(self, name, height):
		self.name = name
		self.height = height

	def walk(self, destination):
		print(self.name, " arrived at ", destination)

person1 = Person("Martin", 1.78)
person2 = Person("Daniela", 1.65)

person1.walk("Budapest")
person2.walk("Prague")

Die unteren zwei Zeilen rufen die Methoden ab, es wir folgendes geprinted:

Martin  arrived at  Budapest
Daniela  arrived at  Prague

statische Methoden

Man kann mit der “@staticmethod”-annotation eine Methode als statisch deklarieren. Solche methoden sind nicht an eine Objekt-Instanz gebunden, sondern an die Klasse. Man kann sie also auch ohne eine Instanz verwenden, sondern direkt über die Klasse.

Jedoch hat sie, logischerweise, keine “self”-Referenz mehr

class Person:

    def __init__(self, name):
        self.name = name

    def get_name(self) -> str:
        return self.name

    @staticmethod
    def hello_world():
        print("hello World")


person1 = Person("Markus")
Person.hello_world() # -> klappt, weil statisch
person1.hello_world() # -> klappt auch immernoch mit Objekt, obwohl statisch

class-Methods

Ist ein edge case. Sind auch statisch aber haben zusätzlich ein “cls” parameter, welcher eine Referenz zur Klasse selber darstellt

class Person:

    def __init__(self, name):
        self.name = name

    @classmethod
    def add_attribute_with_value(cls, attribute, value):
        setattr(cls, attribute, value)
        # cls[attribute] = value


person1 = Person("Markus")
Person.add_attribute_with_value("age", 12)
print(person1.age) # 12
print(Person.age) # 12

Hier wird der Klasse ein Attribut und ein dazugehöriger Wert hinzugefügt. Dies passiert mit der “classmethod”, welche eine Referenz zur Klasse “cls” hat.

Unterschied zwischen Instanz- & Klassen-variablen

Beim oberen Beispiel ist “name” eine Instanz-variabel. Sie wird im init mit self definiert und der jeweiligen Instanz zugewiesen.

Wenn man es jedoch so schreiben würde: 

class Person:
	name = "Martin"
	

person1 = Person()
print(person1.name)

Dann ist “name” eine Klassen-variable. Sie ist nicht mit “self” definiert worden und ist also so im Bauplan definiert. Es hat nun also jede Person den Namen “Martin”

Wenn ich nun also schreiben würde: 

class Person:
    name = ["Martin"]

    def add_last_name(self):
        self.name.append("Bürki")


person1 = Person()
person2 = Person()

person1.add_last_name()

print(person1, person1.name)
print(person2, person2.name)

Bekomme ich folgendes Resultat: 

<__main__.Person object at 0x0000027CFD7215B0> ['Tinu', 'Bürki']
<__main__.Person object at 0x0000027CFD7215E0> ['Tinu', 'Bürki']

→ Obwohl nur bei der “person1”-Instanz der Nachname hinzugefügt wurde, hat nun die person2-Instanz auch den Nachnamen “Bürki”.

Dies ist weil die Methode “add_last_name” sucht bei der Zeile “self.name.append(”Bürki”)”, ob die Instanz einen “name” hat. Da bloss die Klasse den “name” hat, macht es einen Fallback auf die Klassenvariable “name = [”Martin”]”

Da beide Personen Instanzen von derselben Klasse sind, wird nun bei beiden der Nachname hinzugefügt.

Achtung: Wenn ich es jedoch so schreiben würde:

class Person:
    name = "Tinu"

    def add_last_name(self):
        self.name = "Jaqueline"


person1 = Person()
person2 = Person()

person1.add_last_name()

print(person1, person1.name)
print(person2, person2.name)

bekomme ich das: 

<__main__.Person object at 0x00000234EB8315B0> Jaqueline
<__main__.Person object at 0x00000234EB8315E0> Tinu

Der Unterschied ist das Gleichzeichen. Bei “self.name = “Jaqueline”. Denn nun sieht der code, dass zwar keine Instanz-Variable “name” vorhanden ist, macht jetzt jedoch nicht einen Fallback auf die Klassenvariable “name” sondern erstellt einfach direkt eine Instanzvariable.

(Beim vorherigen Beispiel wurde “append” verwendet anstatt “=”. Append setzt nicht etwas gleich, sondern verändert etwas existierendes, macht also einen Fallback)

Vererbung

Klassen können von anderen Klassen erben. Dies geschieht zum Beispiel so: 

class Rectangle:

    def __init__(self, length, width):
        self.length = length
        self.width = width

    def area(self):
        print(self.length * self.width)


class Square(Rectangle):

    def __init__(self, length):
        self.length = length
        super().__init__(self.length, self.length)


my_rect = Rectangle(3, 6)
my_square = Square(3)

my_square.area()
my_rect.area()

Hier erbt die Klasse “Square” von der Klasse “Rectangle”, indem bei der Klasse Square vornedran in Klammer die “parent”-Klasse definiert wird. (class Square(Rectangle))

Dabei ist es wichtig, dass im Konstruktor von “Square” auch der Konstruktor von “Rectangle” ausgeführt wird. Dies wird hier mit dem super().__init__() gemacht. Das Keyword “super()” ist eine Instanz der Elternklasse.

Jetzt kann, wie in der zweitletzten Zeile gezeigt, die Methode area vom Object “my_square” ausgeführt werden, obwohl die Klasse Square keine solche Methode definiert hat. Dies ist eben, weil es nun diese Methode von “Rectangle” geerbt hat. Wenn der Code also in der Square Klasse keine “area”-Methode findet, geht er nun in der/den Elternklasse/n suchen.

Dasselbe gilt auch für Attribute.

Abstrakte Klassen

Abstrakte Klassen sind Klassen, welche Methoden oder Attribute enthalten, welche von anderen Klassen vererbt werden können und überschrieben (personalisiert) werden müssen.

In Python sind sie nicht standardmässig enthalten und werden deswegen auch kaum benutzt.

Dies sieht z.B. so aus:

from abc import abstractmethod


class AbstractMethods:
    def __init__(self):
        pass

    @abstractmethod
    def implement_me(self):
        print("I need to be implemented")


class MyMethods(AbstractMethods):
    def __init__(self):
        super().__init__()

Vererbung von mehreren Klassen

class A:
    def __init__(self):
        print("Constructor of class A")


class B:
    def __init__(self):
        print("Constructor of class B")


class C(A, B):
    def __init__(self):
        A.__init__(self)
        B.__init__(self)


obj = C()

Klasse C vererbt hier von zwei Klassen. Um bei beiden den Konstruktor auszulösen, müssen beide init Funktionen separat ausgeführt werden.
