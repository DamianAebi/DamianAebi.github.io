Monolith

  • Ein Monolith ist ein Projekt dann, wenn Frontend und Backend zusammen im selben Projekt erstellt werden
  • Zu Übungszwecken wurde hier ein Monolith Projekt erstellt, um Kontakte zu speichern und anzuzeigen

Setup

  • Django Projekt erstellen mit virtueller Umgebung (venv)
  • Django gibt uns
    • User-Handling
    • Routing
    • Auth
    • Django-Forms
    • Django-Admin
  • python manage.py runserver
  • .gitignore erstellen:
.idea
.env
.venv
  • Datenbank erstellen (pgAdmin)
  • Packages herunterladen und requirements.txt erstellen: 
pip install django-environ
pip install environ
pip install psycopg2
pip freeze > requirements.txt
  • .venv erstellen:
SECRET_KEY=psssst
DEBUG=True
DB_Name=contacts
DB_USER=postgres
DB_PASSWORD=12345
DB_HOST=localhost
DB_PORT=5432
  • settings-file updaten mit env-variablen: 
import environ

# Intitialize Enviroment-File
env = environ.Env()
environ.Env.read_env('.env')

# SECURITY WARNING: keep the secret key used in production secret!
SECRET_KEY = env('SECRET_KEY')

# SECURITY WARNING: don't run with debug turned on in production!
DEBUG = env('DEBUG') == '1' or env('DEBUG') == 'True'

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql_psycopg2',
        'NAME': env('DB_NAME'),
        'USER': env('DB_USER'),
        'PASSWORD': env('DB_PASSWORD'),
        'HOST': env('DB_HOST'),
        'PORT': env('DB_PORT'),
    }
}

Tables erstellen

# neue user app erstellen fürs Formular
python manage.py startapp user

# tables einfügen 
python .\manage.py migrate

# alternativ können auch migrationen erstellt werden, damit sie später ausgeführt werden können.
# Dabei muss man zuerst eine app erstellen mit models und die App muss im Settings file unter "INSTALLED_APPS" vermerkt werden
# danach kann folgender command ausgeführt werden:
python .\manage.py makemigrations # -> erstellt ein file im migrations-ordner

# nun kann wieder der migrate-befehl ausgeführt werden, um die models zu kreieren.
